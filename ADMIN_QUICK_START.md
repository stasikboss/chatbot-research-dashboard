# מדריך שימוש מהיר - דשבורד ניהול

## גישה ראשונה

### התחברות כ-ADMIN
1. גש ל-http://localhost:3000/dashboard/login
2. התחבר עם:
   - **אימייל**: `researcher@example.com`
   - **סיסמה**: `password123`
3. תראה ניווט נוסף בראש העמוד: "ניהול מערכת", "חוקרים", "התחברויות", "פעילות"

## תפריט ניווט

### עבור חוקרים רגילים (RESEARCHER):
- סקירה
- משתתפים
- ייצוא

### עבור מנהלים (ADMIN) - בנוסף:
- **ניהול מערכת** - סקירה כללית
- **חוקרים** - ניהול משתמשים
- **התחברויות** - היסטוריה
- **פעילות** - לוג

## דפי Admin

### 1. ניהול מערכת (`/dashboard/admin`)
**מה תראה:**
- 4 כרטיסי סטטיסטיקה:
  - סה"כ חוקרים (+ פעילים)
  - פעילים היום (+ כניסות)
  - סה"כ משתתפים (+ הושלמו)
  - שיעור השלמה (+ בתהליך)
- קישורים מהירים לדפים אחרים
- 20 פעולות אחרונות (טיימליין)

**מתי להשתמש:**
- בדיקה מהירה של מצב המערכת
- מעקב אחר פעילות יומית

### 2. חוקרים (`/dashboard/admin/researchers`)
**מה תראה:**
- טבלת כל החוקרים
- חיפוש לפי שם/אימייל
- כפתור "הוסף חוקר חדש"

**עמודות בטבלה:**
- שם
- אימייל
- תפקיד (ADMIN/חוקר)
- סטטוס (פעיל/לא פעיל)
- כניסה אחרונה
- תאריך יצירה
- פעולות (עריכה/סיסמה/מחיקה)

**פעולות:**
- **עריכה** (אייקון עיפרון) - עדכון פרטים
- **סיסמה** (אייקון מפתח) - החלפת סיסמה
- **מחיקה** (אייקון פח) - מחיקת חשבון

### 3. יצירת חוקר חדש (`/dashboard/admin/researchers/new`)
**שדות חובה:**
- שם מלא
- אימייל (ייחודי)
- סיסמה (לפחות 6 תווים)
- תפקיד (ADMIN/חוקר)

**טיפים:**
- מנהלים יכולים לראות ולנהל הכל
- חוקרים יכולים רק לצפות בנתונים
- האימייל חייב להיות ייחודי

### 4. עריכת חוקר (`/dashboard/admin/researchers/[id]/edit`)
**מה אפשר לערוך:**
- שם
- אימייל
- תפקיד (ADMIN ↔ RESEARCHER)
- סטטוס פעיל (✓/✗)

**מה לא אפשר:**
- לשנות תפקיד עצמי
- למחוק עצמך
- למחוק את האדמין האחרון
- לשנות את התפקיד של האדמין האחרון

**שינוי סיסמה:**
- כפתור נפרד "שינוי סיסמה"
- פותח מודל
- דורש 2 פעמים (סיסמה + אימות)

### 5. היסטוריית כניסות (`/dashboard/admin/sessions`)
**מה תראה:**
- טבלת כל ההתחברויות
- פילטרים: תאריך, סטטוס
- סטטיסטיקות: סה"כ, פעילים, הסתיימו

**עמודות:**
- חוקר (שם + אימייל)
- זמן כניסה
- זמן יציאה
- משך זמן
- סטטוס (פעיל/הסתיים)
- מכשיר (דפדפן + OS)
- IP

**פילטרים:**
- סטטוס: הכל / פעיל / הסתיים
- מתאריך / עד תאריך

**טיפים:**
- Sessions פעילים = טרם התנתקו
- משך זמן = רק ל-sessions שהסתיימו
- IP עוזר לזהות כניסה חשודה

### 6. לוג פעילות (`/dashboard/admin/activity`)
**מה תראה:**
- טיימליין של כל הפעולות
- פילטרים: סוג פעולה, תאריך
- פגינציה (50 לעמוד)

**סוגי פעולות:**
- 🟢 כניסה (LOGIN)
- ⚫ יציאה (LOGOUT)
- 🔵 צפייה (VIEW_*)
- 🟠 ייצוא (EXPORT_DATA)
- 🟢 יצירת חוקר (CREATE_RESEARCHER)
- 🔵 עדכון חוקר (UPDATE_RESEARCHER)
- 🔴 מחיקת חוקר (DELETE_RESEARCHER)
- 🟣 שינוי תפקיד (CHANGE_ROLE)
- 🟡 שינוי סיסמה (CHANGE_PASSWORD)

**פילטרים:**
- סוג פעולה (dropdown)
- מתאריך / עד תאריך

**כפתור "טען עוד":**
- טוען 50 פעולות נוספות
- מופיע רק אם יש עוד פעולות

## תרחישים נפוצים

### הוספת חוקר חדש
1. לחץ "חוקרים" בתפריט
2. לחץ "הוסף חוקר חדש"
3. מלא פרטים
4. בחר תפקיד
5. שמור

### שינוי תפקיד מחוקר ל-ADMIN
1. לחץ "חוקרים"
2. מצא את החוקר
3. לחץ על כפתור העריכה (עיפרון)
4. שנה תפקיד ל-"מנהל"
5. שמור

### איפוס סיסמה לחוקר
1. לחץ "חוקרים"
2. מצא את החוקר
3. לחץ על כפתור המפתח
4. הזן סיסמה חדשה (פעמיים)
5. שמור

### מחיקת חוקר
1. לחץ "חוקרים"
2. מצא את החוקר
3. לחץ על כפתור הפח
4. אשר מחיקה
5. החוקר + ההיסטוריה שלו יימחקו

### בדיקת פעילות חשודה
1. לחץ "התחברויות"
2. חפש IP או זמנים חריגים
3. לחץ "פעילות"
4. סנן לפי החוקר החשוד
5. בדוק מה הוא עשה

### צפייה בפעילות שלך
1. לחץ "פעילות"
2. כל הפעולות שלך מתועדות
3. כולל: כניסה, ייצוא נתונים, ניהול משתמשים

## הגנות אבטחה

### מה **לא** אפשר לעשות:
❌ למחוק את עצמך
❌ למחוק את האדמין האחרון
❌ לשנות את התפקיד שלך
❌ להסיר ADMIN מהאדמין האחרון
❌ לגשת ל-`/dashboard/admin` ללא תפקיד ADMIN

### מה **כן** אפשר:
✅ לנהל חוקרים אחרים
✅ לשנות סיסמאות
✅ לשנות תפקידים (אם יש עוד אדמין)
✅ למחוק חוקרים (לא עצמך)
✅ לצפות בכל הפעילות

## טיפים

### ביצועים
- דפי Admin עובדים מהר (React Server Components)
- Pagination בלוג פעילות למניעת עומס
- פילטרים ב-DB level (מהיר)

### אבטחה
- כל פעולה נרשמת
- IP נשמר
- User Agent נשמר
- אין אפשרות לעקוף הגנות

### תחזוקה
- בדוק פעילות חשודה מדי יום
- עקוב אחר חוקרים לא פעילים
- מחק חשבונות ישנים שלא בשימוש

### בעיות נפוצות
**"Forbidden - Admin access required"**
→ אתה לא ADMIN, פנה למנהל

**"Cannot delete the last admin"**
→ צור ADMIN נוסף לפני מחיקה

**"Cannot delete yourself"**
→ בקש ממנהל אחר למחוק אותך

**"Email already exists"**
→ האימייל כבר בשימוש, בחר אחר
